## About

**Doshka** is a portfolio REST API project replicating core Trello functionality. It demonstrates production-ready patterns in modern PHP: JWT authentication, role-based authorization with Voters, async email notifications via a Redis-backed message queue, scheduled cron jobs, and full Docker-based development environment.

## Features

- **JWT Authentication** — register, login, token-based access via `Authorization: Bearer`
- **Boards, Lists & Cards** — full CRUD with drag-and-drop reorder and cross-list card movement
- **Role-Based Access Control** — three board roles (`owner` / `admin` / `member`) enforced via Symfony Voters
- **Card Assignments & Labels** — assign users to cards, attach color-coded labels
- **Activity Log** — per-board audit trail of all actions
- **Async Email Notifications** — board invitations, card assignments, new comments — sent via Symfony Messenger + Redis queue
- **Deadline Reminders** — daily cron at 08:00 UTC via Symfony Scheduler, notifies assignees one day before due date
- **OpenAPI Docs** — Swagger UI auto-generated by API Platform

## Quick Start

**Prerequisites:** Docker & Docker Compose

```bash
# 1. Clone the repository
git clone https://github.com/volodeveth/doshka-backend.git
cd doshka-backend

# 2. Copy environment file
cp .env .env.local

# 3. Start all services
docker compose up -d

# 4. Install PHP dependencies
docker compose exec php composer install

# 5. Generate JWT key pair
docker compose exec php mkdir -p config/jwt
docker compose exec php openssl genrsa -out config/jwt/private.pem -passout pass:doshka_jwt_passphrase 4096
docker compose exec php openssl rsa -pubout -in config/jwt/private.pem -out config/jwt/public.pem -passin pass:doshka_jwt_passphrase

# 6. Run database migrations
docker compose exec php php bin/console doctrine:migrations:migrate --no-interaction
```

The API is now available at **http://localhost/api**

## API Reference

### Authentication

```
POST   /api/auth/register     Register a new user
POST   /api/auth/login        Login → returns JWT token
GET    /api/auth/me           Get current user profile
PATCH  /api/auth/me           Update profile
```

<details>
<summary>Example payloads</summary>

**Register**
```json
{
  "email": "john@example.com",
  "username": "john",
  "password": "secretpassword"
}
```

**Login response**
```json
{
  "token": "eyJhbGciOiJSUzI1NiJ9...",
  "user": { "id": 1, "email": "john@example.com", "username": "john" }
}
```

All subsequent requests require the header:
```
Authorization: Bearer <token>
```
</details>

## Permissions Model

| Action | owner | admin | member |
|---|:---:|:---:|:---:|
| View board & cards | ✅ | ✅ | ✅ |
| Create / edit / delete cards | ✅ | ✅ | ✅ |
| Invite members | ✅ | ✅ | ❌ |
| Change member roles | ✅ | ❌ | ❌ |
| Delete board | ✅ | ❌ | ❌ |
| Edit / delete any comment | ✅ | ✅ | ❌ |

Enforced via **Symfony Voters** (`BoardVoter`, `CommentVoter`).

## Async Email Notifications

Emails are dispatched as messages to a **Redis queue** and consumed by a Messenger worker — fully decoupled from HTTP requests.

| Trigger | Recipient |
|---|---|
| Invited to a board | Invited user |
| Assigned to a card | Assigned user |
| New comment on a card | All card assignees (except commenter) |
| Card deadline is tomorrow | All card assignees |

Deadline reminders are fired automatically via **Symfony Scheduler** (cron `0 8 * * *`, 08:00 UTC).

```bash
# Start the email worker
docker compose exec php php bin/console messenger:consume async -vv
```

Emails can be previewed locally at **http://localhost:8025** (Mailpit).

## Project Structure

```
src/
├── Controller/          # REST API controllers (Auth, Board, List, Card, Label, Comment)
├── Entity/              # Doctrine ORM entities (10 models)
├── Repository/          # Database query repositories
├── Security/
│   ├── BoardVoter.php   # Role-based board authorization
│   └── CommentVoter.php # Comment edit/delete authorization
├── Service/
│   └── ActivityLogger.php  # Explicit activity logging service

├── MessageHandler/      # Email sending handlers
└── Scheduler/
    └── DeadlineReminderTask.php  # Daily 08:00 UTC cron
```
